Pre-submit validation attribution

The form below has three fields with restrictive HTML5 validation. When the scanner fills them with generic synthetic values they trip :invalid state:

  • phone: pattern="\d{10}" — rejects anything with non-digits or wrong length.
  • age: type="number" min="18" max="120" — a filler value outside this range triggers rangeOverflow / rangeUnderflow.
  • zip: minlength="5" — short values trip tooShort.

Pre-CN-3017 the scanner would emit the generic "The form contains validation errors" blurb. Post-CN-3017 it emits per-field attribution with validity signals + value tried + proximity-error text.
