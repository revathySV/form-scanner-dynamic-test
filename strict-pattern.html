Strict-pattern form (guaranteed pre-submit validation failure)

The three fields below have constraints no auto-filler can plausibly satisfy, so the scanner's generated values will reliably trip :invalid and produce a pre-submit failure with three per-field entries.

  • access_code: pattern="PROOF-\d{5}" — must be the literal prefix PROOF- + 5 digits. Generic filler values never start with PROOF-.
  • big_number: type="number" min="50000" max="99999" — filler typically produces small integers.
  • long_field: minlength="64" — filler produces short values like "Test Input".
